# Changelist

## v1.0 — Initial release (2026-06-01)

**UX Fair Dealing Standard v1.0** is the first published version of the standard.

### Standard

- Published the core UXFD checklist (six requirements) for B2C apps:
  1. Show value before asking for money
  2. Make it easy to leave or opt out
  3. Don’t fake urgency, scarcity, or social proof
  4. Be honest and upfront with pricing
  5. Default to user privacy and choice
  6. Give fair warning before new charges
- Documented purpose, adoption steps, and regulatory sources (EU DSA/DFA, deceptive-design taxonomies) in `README.md`.

### Adoption & assets

- Added adoption workflow: self-audit, PR to `apps.md`, badge display, and public declaration.
- Shipped badge and logo assets (`badge_white.png`, `logo.png`, `logo.svg`, `logo_white.svg`).
- Added `apps.md` registry (empty at launch; adopters add entries via PR).

### Accountability

- Defined Hall of Shame process: public reports via PR, adopter vs. non-adopter labeling, right of response, evidence-based merge policy.

### Notes

- `README.md` is the canonical standard text for v1.0; badge links should point to this repo’s README or a `v1.0` tag.
- Breaking or material checklist changes will ship as v2.0+ and be recorded here.
